Architektura aplikacji SPA*

* architektura typowa dla Angular

  • Każdy widok aplikacji posiada kontroler, który zarządza jego stanem.
  • Każdy kontroler odpowiada za:
    • Obsługę akcji (żądań) zainicjowanych przez użytkownika (np naciśnięcie przycisku).
    • Wczytanie danych do zaprezentowania na widoku.
  • Przejście między widokami realizowane jest przez managera stanów (Router).
  • Manager stanów odpowiada za:
    • Przekierowanie do odpowiedniego widoku.
    • Przepisanie ścieżki URL w pasku adresu przeglądaki (tak aby możliwe było wysłanie linku lub dodanie linku do zakładek).
  • Kontroler komunikuje się z serwerem najczęściej wykorzystując w tym celu protokół HTTP/REST.

HTML

HTML to język bazujący na znacznikach służący to tworzenia stron sieci Web.

Przykład dokumentu HTML:

Struktura dokumentu HTML

  • Dokument HTML składa się ze elementów oraz z tekstu.
  • Każdy element zbudowany jest ze znaczników (ang. tags), w szczególności ze znacznika otwierającego i zamykającego.
  • Struktura znaczników musi tworzyć drzewo.
  • W szczególnych przypadkach znacznik zamykający może być pominięty (<img>, <br>, <hr>, itp).

Atrybuty elementów HTML

  • Atrybuty dostarczają dodatkowej informacji dla elementu.
  • Atrybuty muszą być umieszczone w znaczniku otwierającym.
  • Wartość atrybutu może być ujęta w cudzysłowy (") lub apostrofy (').
  • Jeśli wartość atrybutu nie zawiera spacji (białego znaku), cudzysłowy (apostrofy) nie są wymagane.
  • Jeśli wartością atrybutu jest pusty ciąg znaków, można pominąć przypisanie i wymienić jedynie nazwę (jak np w <input required>

Przykłady

  • .intro - wszystkie elementy o klasie intro.
  • #firstname - element o identyfikatorze firstname.
  • p - wszystkie elementy typu p (paragraf).
  • p,div - wszystkie elementy typu p oraz div.
  • div p - wszystkie elementy p zawarte w elemencie div (nie koniecznie bezpośrednio).
  • div>p - wszystkie elementy p zawarte bezpośrednio w elemencie div.
