Architektura klient-serwer

  • Aplikacje klienckie są instalowane niezależnie na maszynach klienckich.
  • Aktualizacja kodu serwera z reguły pociąga za sobą konieczność aktualizacji kodu klienta.
  • Istniały mechanizmy pozwalające na "obejście" tego problemu, jak np. Java Applets, Java Web Start, Flash/ActionScript, wymagały one jednak instalacji dodatkowego oprogramowania (JVM, Pluginy).

Aplikacja webowa

  • Używa standardów takich jak HTML/CSS oraz Javascript, które mogą być wykonywane przez każdą współczesną przeglądarkę internetową.
  • Pozwala na buforowanie kodu po stronie klienta dzięki istnieniu mechanizmów cache.
  • Mechanizmy cache można kontrolować, dzięki czemu twórcy aplikacji mają kontrolę nad aktualizacją klienta razem z serwerem.

AJAX

Technika AJAX w znaczący sposób poprawiła odczucia użytkowników.

Aplikacje webowe stały się bardziej "responsywne" i zaczęły przypominać aplikacje desktopowe.

AJAX został zaadaptowany przez niektóre technologie Server-Side, np JSF.

AJAX - przykład

<!DOCTYPE html>
<html>
<body>
<div id="demo">
<h2>The XMLHttpRequest Object</h2>
<button type="button" onclick="loadDoc()">Change Content</button>
</div>
<script>
function loadDoc() {
  var xhttp = new XMLHttpRequest();
  xhttp.onreadystatechange = function() {
    if (this.readyState == 4 && this.status == 200) {
      document.getElementById("demo").innerHTML =
      this.responseText;
    }
  };
  xhttp.open("GET", "ajax_info.txt", true);
  xhttp.send();
}
</script>
</body>
</html>

Architektura aplikacji Rich Web Application*

* architektura typowa dla Angular

  • Każdy widok aplikacji posiada kontroler, który zarządza jego stanem.
  • Każdy kontroler odpowiada za:
    • Obsługę akcji (żądań) zainicjowanych przez użytkownika (np naciśnięcie przycisku).
    • Wczytanie danych do zaprezentowania na widoku.
  • Przejście między widokami realizowane jest przez managera stanów (Router).
  • Manager stanów odpowiada za:
    • Przekierowanie do odpowiedniego widoku.
    • Przepisanie ścieżki URL w pasku adresu przeglądaki (tak aby możliwe było wysłanie linku lub dodanie linku do zakładek).
  • Kontroler komunikuje się z serwerem najczęściej wykorzystując w tym celu protokół HTTP/REST.
